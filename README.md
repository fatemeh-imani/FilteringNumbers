استفاده از Delegate (Func<int,bool>) به جای Interface

در نسخه قبلی، برای فیلتر کردن اعداد از یک Interface مانند IFilter استفاده می‌شد:

اما در نسخه فعلی، به جای ساخت کلاس‌های متعدد و پیاده‌سازی Interface، از Delegate آماده Func<int,bool> استفاده شده است


Func<int,bool> = یک تابع که عدد می‌گیرد و نتیجه‌ی true/false برمی‌گرداند.

می‌توان هم متد جداگانه به آن داد (IsMatch) و هم Lambda مستقیم نوشت


##مزایای استفاده از Delegate

سادگی و کوتاه‌تر بودن کد

نیاز به تعریف کلاس یا Interface جدید برای هر شرط نیست.

انعطاف‌پذیری بالا

می‌توان شرط‌ها را به صورت Lambda یا متدهای جداگانه داد.

قابلیت استفاده مجدد

می‌توان چند شرط مختلف را بدون تغییر متد FilterNumbers اعمال کرد.

خوانایی بهتر

کد مستقیماً مشخص می‌کند که شرط چیست، بدون نیاز به مراجعه به کلاس‌های مختلف.


##معایب و محدودیت‌ها  

کاهش ساختار کلاسیک OOP

اگر پروژه بزرگ شود و بخواهیم شرط‌ها را در کلاس‌ها سازمان‌دهی کنیم، ممکن است مدیریت آنها کمی سخت‌تر شود.

محدودیت در ترکیب شرط‌ها

اگر بخواهیم شرط‌های پیچیده یا سلسله‌مراتبی داشته باشیم، Lambda‌های طولانی ممکن است خوانایی را کم کنند.

عدم امکان استفاده از polymorphism مستقیم

در Interface می‌توانستیم از ارث‌بری و پیاده‌سازی‌های مختلف استفاده کنیم، اما با Delegate فقط تابع ارسال می‌کنیم.
